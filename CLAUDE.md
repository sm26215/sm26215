## Structure

- `README.md` — The sole content file. Its rendered Markdown appears as the GitHub profile landing page.

## Conventions

- **No build system, dependencies, or tests.** This is a static Markdown-only repo.
- Edits to `README.md` are deployed instantly when pushed to `main` (GitHub renders it automatically).
- The README uses GitHub-Flavored Markdown (GFM) with HTML comments for template hints.

## Development Workflow

1. Edit `README.md` directly.
2. Commit to `main` and push — changes appear on the profile immediately.
3. No CI/CD pipeline, linters, or pre-commit hooks are configured.

## Guidelines for AI Assistants

- Keep changes focused on `README.md` unless the user explicitly requests new files.
- Preserve the user's personal tone and style when editing profile content.
- Do not remove existing content without explicit permission.
- GitHub profile READMEs support GFM features: badges, images, HTML blocks, and GitHub Stats widgets.
